# AGENTS.md

This repository is not a startup website, portfolio template or promotional platform.

It is a long-term contemporary artistic platform for Felipe Rocha.

The project operates between:

* artistic practice
* film
* sound
* listening
* cultural direction
* archive
* editorial systems
* institutional presence

The website must always prioritize:

* silence
* reduction
* spatial rhythm
* editorial clarity
* permanence
* calm pacing
* negative space
* restrained interaction

The website must NEVER feel:

* startup-like
* tech-oriented
* commercially aggressive
* portfolio-heavy
* trend-driven
* overstimulating
* social-media-inspired

Every design decision should answer:

"Does this reduce perceptual anxiety or increase it?"

If it increases stimulation:
remove it.

---

## Core structure

Main sections:

* Projects
* Practice
* Archive
* Press
* Contact

DESCANSO is a separate internal territory.

DESCANSO must feel slower, quieter and more reduced than the main platform.

---

## Typography

Only use:

* Inter
* Inter Mono

Avoid:

* expressive typography
* decorative typography
* excessive font weights

---

## Color system

Allowed:

* off-white
* charcoal
* mineral gray
* soft black

Avoid:

* gradients
* vibrant colors
* high contrast
* branding aesthetics

---

## Motion

Allowed:

* opacity fades
* restrained hover transitions
* slow transitions

Avoid:

* parallax
* complex motion
* exaggerated smooth scrolling
* tech-like animation

---

## Images

Images should feel:

* physical
* documentary
* quiet
* spatial
* restrained

Avoid:

* instagram aesthetics
* lifestyle aesthetics
* wellness imagery
* overproduction

Use existing image formats:

* hero (16:9)
* documentation (3:2)
* vertical (4:5)
* thumb (1:1)

---

## Text

Text should:

* create space
* suggest
* remain restrained
* avoid overexplanation

Avoid:

* branding copy
* startup language
* excessive philosophy
* academic density
* motivational tone

---

## Technical constraints

Keep:

* static HTML
* lightweight CSS
* lightweight JS
* GitHub Pages compatibility

Do not:

* introduce frameworks
* add React
* add unnecessary dependencies
* create heavy JS architecture

---

## Bilingual system

Maintain:

* single bilingual website
* .lang-en
* .lang-pt

Do not:

* create separate PT/EN websites

English remains default.

---

## Before making large structural changes

Always:

1. analyze current architecture
2. preserve editorial rhythm
3. preserve silence and whitespace
4. avoid increasing visual density
5. prefer reduction over addition
